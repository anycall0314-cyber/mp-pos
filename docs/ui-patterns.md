# UI 版型規約

> 對應 [decisions.md](decisions.md) 內「UX 模式 — 三種頁面版型」決策。所有業務頁面都應落入下列三種版型之一,不要混搭。

## 為什麼需要三種版型

舊系統「歐睿手機玩家 / 歐睿創意POS」在實務上是兩個介面分工：

- **後台（IE 桌面程式）**：錄入單據（進貨/銷貨/調撥/維修),鍵盤為主,以「資料錄入效率」為導向
- **前台（網頁）**：100+ 個查詢報表,以「資料檢索與輸出」為導向

新系統把這兩種模式 + 主檔維護合併到同一個 React app,但 UI 版型必須分清楚,否則：

- 用 Master-Detail 套錄入頁 → 業務員一筆銷貨要點 30 次滑鼠,慢
- 用 Master-Detail 套報表頁 → 沒篩選列,看不到日期區間/倉庫切片
- 用錄入頁版型套主檔 → 用滑鼠瀏覽商品清單變得困難

## 三種版型總覽

| 版型 | 用途 | 特徵 |
|------|------|------|
| [Master-Detail](#master-detail) | 主檔維護 | 左列表 + 右詳情,鍵鼠混用 |
| [EntryForm](#entryform) | 開單據 | 上方單頭 + 中央明細,鍵盤連續錄入 |
| [ReportPage](#reportpage) | 查詢報表 | 上方篩選 + 中央結果 + 下方總計與匯出 |

## Master-Detail

主檔資料維護（商品、類別、倉庫、供應商、客戶等）。已有實作於 `frontend/src/components/master-detail/MasterDetail.tsx`。

### 版型

```
┌──────────────────────────┬────────────────────────────────┐
│ 篩選列                    │ 詳情頁籤（基本 / 序號 / 歷史） │
│ [搜尋框] [類別▾] [啟用▾]  ├────────────────────────────────┤
├──────────────────────────┤                                │
│ 列表表格                  │ 詳情內容                       │
│ ▷ A001 ...                │ 品號:     ...                  │
│ ▷ A002 ...                │ 品名:     ...                  │
│ ▶ A003 ...  ← 選中         │ 規格:     ...                  │
│ ▷ A004 ...                │ ...                            │
│                          ├────────────────────────────────┤
│                          │ [編輯] [刪除] [儲存]            │
└──────────────────────────┴────────────────────────────────┘
```

### 規約

- 左欄寬度可拖,預設 380px,記在 localStorage
- 列表表格用虛擬滾動（資料量大時）
- 詳情頁籤至少 1 個,最多 6 個
- 右下角操作列按鈕固定底部
- 點選左欄列即時切換右欄,不需「載入」按鈕
- 適用頁面：商品 / 類別 / 倉庫 / 供應商 / 客戶 / Carrier / TelecomPlan / Brand 等所有主檔

## EntryForm

開單據（進貨、銷貨、調撥、借出、維修單）。**業務員每天用得最頻繁,效率最關鍵**。

### 版型

```
┌────────────────────────────────────────────────────────────┐
│ 工具列:[F2 編輯] [F5 過帳] [F8 刪除] [F10 存檔] [ESC 取消] │
├────────────────────────────────────────────────────────────┤
│ 單頭:                                                       │
│ 單號  PO-000001 (自動)      供應商 [SUP001 蘋果經銷    ▾]  │
│ 倉庫  [TPE-01 台北旗艦店▾]  日期   [2026-05-17]            │
│ 業務員 [JB00 蔡樂雅      ▾]  備註  [____________]           │
├────────────────────────────────────────────────────────────┤
│ 明細:                                                       │
│ # │品號     │品名         │數量│單價   │金額   │IMEI 清單  │
│ 1 │PH-000001│iPhone 15 Pro│ 3 │27,000 │81,000 │[掃碼...]  │
│ 2 │PH-000002│iPhone 15 Pro│ 2 │27,500 │55,000 │[掃碼...]  │
│ 3 │[輸入 SKU 自動帶...]                                    │
├────────────────────────────────────────────────────────────┤
│ 附加費用:[3,000]   貨款小計:136,000  總額:139,000          │
└────────────────────────────────────────────────────────────┘
```

### 規約

- **鍵盤導航**：所有 input 用 `tabIndex` 從上到下排好,Tab/Shift-Tab 移動,Enter 進入下一格
- **F-key 工具列**：F2 編輯 / F5 過帳 / F8 刪除目前列 / F10 存檔 / ESC 取消當下動作回上一層
- **商品快查**：在「品號」格輸入 SKU 開頭或品名,顯示下拉建議,Enter 帶資料 + 自動跳到「數量」
- **條碼掃描連續輸入**：「IMEI 清單」欄位 focus 時,掃一台→自動寫入 + 自動到下一個空格,掃滿 `qty` 自動跳下一行
- **金額自動算**：unit_price × qty = amount,即時更新
- **過帳鍵 (F5)**：彈出確認 modal「確定過帳?過帳後不能修改」,確認後鎖定整單
- **單頭與明細的視覺分區**：用 border 與背景區分,讓眼睛不會在 30 個欄位裡迷路
- **適用頁面**：進貨單 / 銷貨單 / 銷退單 / 調撥單 / 借出單 / 維修單 / 中古機收購單

### 銷貨單的電信欄位

銷貨明細的電信欄位（SIM / 門號 / 方案 / 佣金 / 上線日）**預設折疊**,商品需要綁門號時點「展開」才出現,避免一般銷貨被欄位淹沒。

## ReportPage

查詢報表。對應舊系統 100+ 個編號報表（MM/SA/FI/SIM 等）。

### 版型

```
┌────────────────────────────────────────────────────────────┐
│ 報表名稱: MM020 IMEI 明細表                                │
├────────────────────────────────────────────────────────────┤
│ 篩選:                                                       │
│ 日期區間 [2026-04-01] 至 [2026-05-17]                       │
│ 倉庫     [全部              ▾]                              │
│ 類別     [全部              ▾]                              │
│ 狀態     [☑在庫 ☐已售 ☐維修中 ☐作廢]                       │
│ 序號     [_____________________]                            │
│                                       [清除] [查詢]         │
├────────────────────────────────────────────────────────────┤
│ 結果 (273 筆):                                              │
│ IMEI         │SKU      │品名           │倉   │狀態│進貨日  │
│ 358120123456 │PH-000001│iPhone 15 Pro  │TPE  │在庫│04-12   │
│ 358120123457 │PH-000001│iPhone 15 Pro  │TPE  │在庫│04-12   │
│ ...                                                         │
├────────────────────────────────────────────────────────────┤
│ 總計:273 筆   採購總成本:$7,651,200      [CSV] [Excel]     │
└────────────────────────────────────────────────────────────┘
```

### 規約

- **報表編號**：每張報表都掛一個編號（沿用 MM/SA/FI 或新編 R001+）,顯示在標題列
- **Command Palette (Cmd+K)**：全站快捷,輸入編號或名稱片段直接跳轉
  - 例:輸入「MM020」、「IMEI 明細」、「庫存異動」都能跳到對應報表
- **篩選列**：collapse 可隱藏,焦點預設在第一個篩選欄位
- **結果表**：虛擬滾動,欄位可拖排序,可點欄位標頭排序,可雙擊欄位邊界調整寬度
- **匯出**：至少支援 CSV;Excel 走 server 端產出 xlsx
- **儲存查詢**：常用篩選組合可命名儲存（例:「本月在庫 iPhone」）, 第二次點直接帶條件
- **多維度切片（SA040 風格）**：報表可有「分組依據」選單,例如毛利彙總可 by 日期/客戶/業務員/倉庫
- **適用頁面**：所有 MM/SA/FI/SIM 編號報表

### 報表清單管理

新系統的報表清單應在資料庫管理（不寫死在前端 routes 裡）,這樣能：

- 業務員可看到「我有權限的所有報表」
- 老闆可控制誰能看哪份報表（對應 SEC005 報表權限設定）
- 新增報表時不用改前端代碼

對應 model 設計（Phase 4 加入）：

- `auth.ReportDefinition`：code（MM020 等）、name、category、route_path、required_permission
- `auth.ReportPermission`：role × report_definition × can_view

## 共通約定

### 顏色與裝飾

- 深色版型（dark.css 為主）,白色版型為輔
- **禁用** emoji 與其他裝飾性符號（見 [memory feedback-no-emoji]）
- icon 只用功能性（搜尋、過帳、刪除、匯出),不用情緒性（笑臉等）

### 鍵盤快捷鍵全站約定

| 快捷鍵 | 功能 |
|--------|------|
| Cmd+K | 開啟 Command Palette |
| Cmd+S | 儲存當前頁面 |
| Cmd+F | 在當前頁面內搜尋 |
| Esc | 取消當下動作 / 關閉 modal |
| F2 | 編輯選中項 |
| F5 | 過帳（限 EntryForm） |
| F8 | 刪除選中項 |
| F10 | 存檔（限 EntryForm） |
| / | 聚焦到搜尋框 |

### 響應式

- 桌面為主（≥ 1280px）
- 不主動支援手機,但要確保 iPad 橫向（1024px）可用
- 手機體驗放 Phase 5 SaaS 化或行動 APP 另開
